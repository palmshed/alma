# Palmshed Design Language

Palmshed products are designed to feel quiet, timeless, and dependable. The interface
should never compete with the user's work. Every decision should reduce visual noise
rather than add personality.

## Principles

### Quiet

Avoid loud gradients, oversized buttons, excessive color, floating decorations, or
unnecessary animation.

The interface should almost disappear.

---

### Calm

Spacing is generous.

Animations are slow and subtle.

Nothing should flash, bounce, or demand attention.

---

### Functional

Every visual element must communicate purpose.

If an element does not improve usability, remove it.

---

### Consistent

All Palmshed products should feel like they belong to one family.

Reuse:

- icons
- spacing
- corner radius
- typography
- motion
- colors

Do not invent a new visual language for each application.

---

### Minimal, not empty

Minimal does not mean removing functionality.

It means removing distraction.

Whitespace should improve focus, not create emptiness.

---

### Typography first

Text is the primary interface.

Hierarchy comes from:

- size
- weight
- spacing

Not from excessive color.

---

### Motion

Animation should communicate state.

Examples:

- page transition
- loading
- hover
- expanding panels

Never animate purely for decoration.

---

### Color

Color communicates state.

Not decoration.

Green indicates success or active state.

Muted grays form the interface.

Avoid multiple accent colors.

---

### Icons

Icons support labels.

Icons should rarely appear without accompanying text unless universally understood.

Use the Glimpse icon library consistently.

---

### Components

Every component should feel reusable.

If a button or card could belong in another Palmshed product without modification,
it is probably designed correctly.

---

### Goal

The user should notice their work.

They should not notice the interface.

If someone remembers the UI instead of what they accomplished with it, the interface
is too loud.
